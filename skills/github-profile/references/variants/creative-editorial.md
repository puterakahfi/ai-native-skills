# Variant — Creative Editorial

Use for designer-engineers, creative technologists, product thinkers, and multi-disciplinary builders whose authored voice is part of the real identity.

## Direction Contract

```yaml
variant: creative-editorial
density: balanced
alignment: mixed-editorial
expression: expressive-but-controlled
containment: none-or-selective
separators: brand-dependent
proof_emphasis: medium
thesis_emphasis: high
asset_policy: restrained-owned-assets
widget_policy: none-by-default
```

## Foundation Expression

```text
hierarchy       authored statements create local peaks without competing anchors
grouping        recurring editorial anatomy keeps proof understandable
alignment       deliberate centered, left, or indented moments with stable anchors
rhythm          narrative pauses alternate with compact proof regions
balance         expressive moments are counterweighted by readable semantic text
flow            identity → authored premise → selected proof → method → invitation
legibility      expression never lowers text size, contrast, or reading continuity
```

## Recognition Device

Choose one coherent expressive device:

```text
authored section vocabulary
one recurring sentence pattern
one semantic system diagram
one owned visual motif
one deliberate mixed-alignment rhythm
```

Do not combine all devices. Creativity comes from coherent authorship, not visual randomness.

## Hero

A strong creative-editorial opening may use:

```text
name or wordmark
specific multi-disciplinary positioning
one authored thesis or tension
one concise explanation grounded in real work
one action bridge
```

The thesis must be specific enough that it could not be pasted onto an unrelated profile.

## Selected Work

Projects may use more authored labels than a conventional professional profile, but must still communicate:

```text
what it is
who or what it serves
what the person owns
why the work matters
where the evidence lives
```

Example anatomy:

```markdown
### 01 · [Project Name →](LINK)

**The question** — the problem or tension being addressed.  
**The work** — role, contribution, and system built.  
**The evidence** — verified state, product, repository, or case study.
```

Numbering is optional and should not be added when it conflicts with the brand.

## Section Language

Intentional section naming can differentiate the profile:

```text
Selected systems
What I am testing
Principles in practice
Current field notes
Open paths
```

Each label must remain understandable. Poetic ambiguity must not replace information architecture.

## Visual Assets

Allowed only when stable and meaningful:

```text
verified personal wordmark
repository-owned diagram
consistent project image set
theme-aware editorial graphic
```

Nearby semantic text must preserve meaning when the asset fails.

## Brand Compatibility

For a calm or zen brand:

```text
use authored language and cadence as the creative device
keep containment and separator noise low
avoid neon, emoji walls, random ASCII, and animation
preserve purposeful whitespace rather than theatrical emptiness
```

For a more expressive brand:

```text
allow one stronger asset or accent system
retain one stable editorial rhythm across all sections
keep proof and actions visually recoverable
```

## Prohibited Defaults

```text
mixed emoji and icon families
ASCII art without semantic value
typing animation as identity
random per-section alignment
multiple unrelated visual motifs
critical claims inside images only
creative copy that hides project ownership
unverified logos, clients, or metrics
```

## Failure Signals

```text
visitor remembers the styling but not the work
section names are clever but unclear
proof anatomy changes unpredictably per project
narrow width destroys the intended sequence
creative expression conflicts with brand identity
visual assets dominate semantic content
```

## Review Focus

Prioritize message specificity, authored voice, hierarchy peaks, consistency of the expressive device, proof clarity, narrow-width flow, and brand compatibility.
