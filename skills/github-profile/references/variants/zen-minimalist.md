# Variant — Zen Minimalist

Use for calm personal brands where authorship, restraint, and purposeful space are stronger than decorative assets.

## Direction Contract

```yaml
variant: zen-minimalist
density: sparse-but-information-bearing
alignment: centered-opening-then-left-reading
expression: restrained
containment: none
separators: none-by-default
proof_emphasis: medium
thesis_emphasis: high
asset_policy: no-asset-by-default
widget_policy: none
```

## Foundation Expression

```text
hierarchy       role scale, weight, measure, and placement
 grouping        proximity; no border required
alignment       one bounded centered opening; detailed reading left-aligned
rhythm          element gap < item gap < section gap
balance         whitespace frames content but does not strand it
flow            identity → proof → thesis → capability → focus → action
legibility      normal semantic text; no tiny critical labels
```

## Hero

The opening should be quiet but specific:

```text
name
role or positioning
one authored thesis
one concise supporting explanation
bounded navigation or action
```

Avoid a decorative banner, greeting, generic quote, or long autobiography.

## Selected Work

Group each project through proximity:

```markdown
#### [Project →](LINK)

One-sentence purpose.

*Role · status · evidence type*

Concise meaningful contribution or verified outcome.
```

Do not simulate cards with blockquotes, tables, borders, code fences, or horizontal rules when the brand is line-free.

## Typography and Voice

```text
calm, direct, authored
short sentences
limited bold emphasis
italic reserved for metadata or quiet nuance
section headings remain visible and semantic
```

Minimalist does not mean small type, weak contrast, or low-information labels.

## Space Contract

Every large interval must be classified as:

```text
grouping | pacing | framing | narrative | utility
```

Reduce any interval without a named role. Sparse content usually needs less space, not more.

## Brand Compatibility

Strong fit:

```text
zen
Ma-led
quiet editorial
minimal personal identity
calm technical authority
```

Possible adaptation:

```text
left-aligned opening when the brand is fully editorial
subtle owned wordmark when already verified
one restrained accent image when essential to recognition
```

## Prohibited Defaults

```text
cardification
blockquote project modules
horizontal dividers
code boxes as decoration
badge rows
stats widgets
visitor counters
tiny uppercase labels as sole section identity
repeated large <br> stacks
```

## Failure Signals

```text
content mass floats in a tall empty canvas
projects detach from their metadata
all text roles have nearly equal visual weight
hero feels generic because the thesis is interchangeable
whitespace slows reading instead of pacing it
minimalism hides proof or useful actions
```

## Review Focus

Prioritize hierarchy, proximity, spatial rhythm, balance, reading measure, first-viewport specificity, and narrow-width continuity.
