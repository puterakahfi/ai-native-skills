# Copywriting

## When to Load

Load when:
- Writing or auditing any hero headline, stance, bio, or section heading
- Content feels generic, vague, or could apply to anyone
- User asks "is this copy good?"
- Producing any landing page, portfolio, or product page

---

## The Content Hierarchy (top → bottom)

Every piece of content answers one question at a time, in order:

```
1. WHO ARE YOU?          → Name, role — immediate, no jargon
2. WHAT DO YOU DO?       → One sentence, concrete outcome — not process
3. WHY DOES IT MATTER?   → Proof, specificity, stakes
4. WHAT CAN I SEE?       → Work, evidence, deliverables
5. HOW DO I REACH YOU?   → Contact — minimal friction
```

If the page skips a level or reorders these → content hierarchy broken.

## The Value Proposition Formula

```
[WHO] helps [AUDIENCE] [ACHIEVE OUTCOME] by [METHOD/DIFFERENTIATOR]

Collapsed for personal portfolio:
  [ROLE] who [DIFFERENTIATOR] — [PROOF POINT]

Example (bad):
  "Full stack developer passionate about building great products"
  → generic, could be anyone, no proof

Example (good):
  "Full stack engineer. I design systems that are still maintainable five years later."
  → specific claim (maintainability), time anchor (5 years), implies longevity bias

Test: can this sentence apply to 1000 other people? If yes → rewrite.
```

## Headline Formulas

### H1 — Identity Anchor
```
Formula: [Name]. [Role] — nothing else.
Purpose: establish who, not what. Work establishes what.
Length:  1–3 words after name. Not a sentence.

Bad:  "Putera Kahfi — Full Stack Engineer Building the Future"
Good: "Putera Kahfi."  (name is enough — work section carries the rest)
```

### H2 — Section Navigational
```
Formula: [Gerund or noun] — directional, ≤ 4 words
Purpose: tell user what this section is, not what to feel
Length:  ≤ 4 words

Bad:  "Some of My Best Work" (hedging), "Explore My Projects" (imperative without CTA)
Good: "Selected work", "About", "Let's talk."
```

### H2 — Statement (manifesto/about)
```
Formula: [Specific claim] — concrete, falsifiable, not inspirational fluff
Purpose: establish worldview, create resonance with right audience
Length:  ≤ 8 words, 1–2 lines

Bad:  "I build things that matter" (vague, unfalsifiable)
Good: "One codebase. Five products." (specific, countable, implies architectural decision)
```

### Stance / Bio (supporting paragraph)
```
Formula: [What you do] + [How you think] + [What that means for work]
Length:  2–3 sentences max. ≤ 45 words.
Tone:    Direct, first person, no hedging ("I try to", "I hope to", "passionate about")

Bad:  "I'm a passionate developer who loves to learn and build innovative solutions."
Good: "Full stack engineer. I design systems that are still maintainable five years later —
       modular, bounded, honest about complexity."
       → specific domain (systems), specific claim (maintainability), characteristic style (3 adjectives)

Checklist:
  □ No "passionate", "love", "innovative", "cutting-edge", "synergy"
  □ No hedging verbs ("try to", "aim to", "hope to")
  □ At least one specific/concrete detail (number, named concept, named product)
  □ ≤ 45 words
  □ Could be said out loud without sounding corporate
```

## Microcopy Rules

### Labels and Tags
```
□ ≤ 2 words per label
□ Noun or gerund — no articles ("the", "a")
□ Consistent grammatical form (all nouns OR all gerunds, not mixed)
□ All caps for meta/decorative labels — title case for navigational labels
```

### CTA (if present)
```
Formula: [Verb] + [Object] — specific action
Bad:  "Learn More", "Click Here", "Submit"
Good: "Read the post", "View Blueprint", "Send a message"
```

### Live badge / status
```
□ One word: "Live", "Beta", "Archived"
□ Paired with visual signal (green dot for Live)
□ Never "Currently Live" or "Now Live" — redundant
```

### Contact section heading
```
□ Conversational, not corporate
□ Implies easy access, not a formal process
Bad:  "Get in Touch", "Contact Me"
Good: "Let's talk.", "Say hello.", "hi@pkahfi.com" (email itself as heading)
```

## Tone Calibration

### Tone Axes (rate each 1–5)
```
Formal ←————————→ Conversational
Serious ←————————→ Playful
Abstract ←————————→ Concrete
Hedged ←————————→ Confident
Generic ←————————→ Specific
```

### Portfolio / Personal Landing Page Target
```
Formal:         2 (lean conversational — this is a person, not a company)
Serious:        3 (balanced — professional but not stiff)
Abstract:       1 (maximum concrete — specificity = credibility)
Hedged:         1 (maximum confident — no hedging)
Generic:        1 (maximum specific — every sentence falsifiable)
```

### Tone Audit — Red Flags
```
□ Any sentence that starts with "I am passionate about..."
□ Any use of "innovative", "cutting-edge", "world-class", "best-in-class"
□ Any sentence that could be copy-pasted to another person's portfolio unchanged
□ Longer than 3 sentences in a single paragraph on a landing page
□ Vague nouns: "things", "solutions", "stuff", "projects" without specifics
```

## Content Density Gates

```
Gate CP1: Value Proposition Clarity
  □ H1 + stance together answer WHO + WHAT DO YOU DO in ≤ 3 seconds
  □ Stance is ≤ 45 words
  □ No generic phrases (passionate, innovative, etc.)
  Score: __ / 10

Gate CP2: Specificity
  □ At least 1 concrete detail per section (number, named concept, named product)
  □ No vague nouns without qualifiers
  □ Work entries have title + description that differs from the title
  Score: __ / 10

Gate CP3: Microcopy Consistency
  □ Labels: consistent grammatical form
  □ CTAs: verb + object (if present)
  □ Status badges: one word
  Score: __ / 10

Gate CP4: Tone Consistency
  □ No hedging verbs
  □ No buzzwords
  □ Conversational but not casual
  Score: __ / 10
```
